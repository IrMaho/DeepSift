# راهنمای جامع استفاده از DeepSift CLI 🚀

موتور **DeepSift** یک سیستم پیشرفته برای مدیریت دانش، جستجوی معنایی (Semantic Search) و درک ساختار کد (Graphify) است که تماماً به صورت لوکال (آفلاین) اجرا می‌شود. این ابزار به شما (و هوش مصنوعی) کمک می‌کند تا روی پروژه‌های عظیم، مستندات زبان‌ها (مثل فریم‌ورک فلاتر) و مهارت‌های کاستوم به صورت ۱۰۰٪ مسلط شوید.

در این مستند، تمام دستورات کلیدی را با مثال‌های واقعی برای استفاده روزمره بررسی می‌کنیم.

---

## ۱. مدیریت قلمروهای دانش (Realms) 🌍

شما می‌توانید برای هر نوع داده‌ای یک **قلمرو (Realm)** جداگانه بسازید تا جستجوهای شما دقیق و تفکیک‌شده باشد. مثلاً یک قلمرو برای کد پروژه، یکی برای مستندات رسمی فلاتر، و یکی برای مهارت‌ها.

### اضافه کردن یک قلمرو جدید
فرض کنید می‌خواهید مستندات فلاتر را که در پوشه `flutter_docs` دانلود کرده‌اید، به سیستم معرفی کنید:
```bash
deepsift realm add flutter_docs --type docs --source ./flutter_docs
```
- `--type`: می‌تواند `code` (برای سورس‌کد)، `docs` (برای مستندات متنی/مارک‌داون) یا `skill` (برای دستورالعمل‌های ایجنت) باشد. پارسر (Parser) بر این اساس رفتار متفاوتی برای تکه‌تکه کردن فایل‌ها (Chunking) خواهد داشت.
- `--source`: آدرس پوشه‌ای که می‌خواهید ایندکس شود (می‌تواند آدرس نسبی یا مطلق باشد).

**مثال‌های دیگر:**
```bash
# اضافه کردن یک ریپازیتوری کدهای بک‌اند به عنوان سورس کمکی
deepsift realm add backend_api --type code --source ../backend_repo

# اضافه کردن مهارت‌های هوش مصنوعی
deepsift realm add ai_skills --type skill --source ./skills

# استفاده از سورس‌کد یک فریم‌ورک/پروژه خارجی (مانند سورس فلاتر) به عنوان مرجع یا مستندات در پروژه فعلی
deepsift realm add flutter_src --type code --source C:\src2\flutter
```

### مشاهده و حذف قلمروها
```bash
# لیست کردن تمام قلمروهای متصل به سیستم
deepsift realm list

# حذف یک قلمرو در صورت عدم نیاز
deepsift realm remove backend_api
```

## ۲. ایندکس کردن داده‌ها (Indexing) 🗂️

بعد از اضافه کردن یک قلمرو، باید آن را ایندکس کنید تا دیتابیس برداری (Vector DB) آن ساخته شود.

```bash
# ایندکس کردن قلمرو جدیدی که ساختید (مثلاً مستندات فلاتر)
deepsift index --realm flutter_docs

# ایندکس کردن سورس فلاتر به عنوان رفرنس خارجی
deepsift index --realm flutter_src

# ایندکس کردن قلمرو پیش‌فرض پروژه (کدهای اصلی)
deepsift index --realm code

# بازسازی کامل ایندکس از صفر (در صورت بروز مشکل یا تغییرات بسیار حجیم)
deepsift index --force --realm code
```
*نکته: دستور `deepsift index` به صورت افزایشی (Incremental) کار می‌کند؛ یعنی فقط فایل‌های جدید یا تغییر یافته را بررسی می‌کند که سرعت بسیار بالایی دارد.*

---

## ۳. جستجوی پیشرفته (Semantic Search) 🔍

وقتی قلمروها ایندکس شدند، می‌توانید با پرس‌وجوی طبیعی (زبان انسان) داخل آن‌ها جستجو کنید.

```bash
# جستجوی یک مفهوم در کدهای پروژه (قلمرو پیش‌فرض)
deepsift search "how is user authentication implemented?"

# جستجوی یک مفهوم داخل مستندات فلاتر
deepsift search "state management using Riverpod" --realm flutter_docs

# جستجوی همزمان داخل کدها، مستندات و مهارت‌ها!
deepsift search "how to handle login errors" --all-realms

# جستجوی چندتایی همزمان (Batch Search) برای سرعت بیشتر
deepsift search "login logic" "database connection" "error handling"
```

## ۴. جستجوی مقایسه‌ای و تحلیلی (Vector Diff Search) ⚖️

یکی از قدرتمندترین ابزارهای DeepSift دستور `compare` است. این دستور به شما اجازه می‌دهد کدهای یک پروژه را با مستندات یا یک پروژه دیگر مقایسه کنید تا **مغایرت‌ها (Gaps)** یا **شباهت‌ها (Similarities)** را پیدا کنید.

**کاربرد واقعی:** آیا کدهایی که برای احراز هویت (Auth) نوشته‌ام، با استانداردها و مستندات رسمی فلاتر تطابق دارد یا چیزی را فراموش کرده‌ام؟

```bash
deepsift compare code flutter_docs --query "authentication"
```
**خروجی این دستور چه خواهد بود؟**
سیستم ابتدا کدهای شما را پیدا می‌کند، سپس همان کدها را به عنوان الگو روی مستندات فلاتر جستجو می‌کند:
- **🔄 SIMILARITIES:** کدهایی که امتیاز تطابق بالایی (>= 0.45) با مستندات دارند (نشان می‌دهد پیاده‌سازی شما استاندارد است).
- **⚠️ GAPS:** کدهایی در پروژه شما که هیچ معادل قدرتمندی در مستندات ندارند (ممکن است یک روش منسوخ‌شده استفاده کرده باشید یا بخشی از مستندات را پیاده نکرده باشید).

## ۵. ابزارهای معماری و تحلیل ساختار (Architecture & DNA) 🧬

گاهی جستجوی کلمات کافی نیست و نیاز دارید معماری کل پروژه را درک کنید.

```bash
# استخراج نقشه معماری پروژه (کلاس‌های اصلی، روابط و God Node ها)
deepsift arch

# پیدا کردن وابستگی‌ها (چه فایل‌هایی از این کامپوننت استفاده کرده‌اند؟)
deepsift deps "src/components/Button.tsx"

# استخراج DNA و هوش محیطی پروژه (شامل دیزاین‌سیستم، توکن‌ها و قوانین کانونشن)
deepsift dna
```

## ۶. تنظیمات و تاریخچه 🛠️

```bash
# منوی تعاملی برای انتخاب اینکه چه پوشه‌هایی در قلمرو code نادیده گرفته شوند (مثل node_modules)
deepsift config

# مشاهده وضعیت دیتابیس (تعداد فایل‌ها و چانک‌ها)
deepsift status

# پاکسازی کامل تاریخچه جستجوها و دیتابیس
deepsift clean
```

### جریان کاری (Workflow) پیشنهادی برای شروع یک پروژه جدید:
1. ترمینال را در پوشه پروژه باز کنید.
2. با `deepsift config` پوشه‌های اضافی را Exclude کنید.
3. کدها را ایندکس کنید: `deepsift index --realm code`
4. مستندات یا ابزارهای جانبی را اضافه کنید: `deepsift realm add my_docs --type docs --source ./docs` و سپس `deepsift index --realm my_docs`.
5. از قدرت بی‌نهایت `deepsift search` و `deepsift compare` لذت ببرید!
